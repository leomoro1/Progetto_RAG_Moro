# Local RAG System for Technical Manual Retrieval in UAV Operations

An isolated, secure, and fully standalone Retrieval-Augmented Generation (RAG) pipeline designed for Unmanned Aerial Vehicle (UAV) operators. This system indexes the official **DJI Mini 2 Technical Manual** into a local vector store, allowing pilots to query complex flight procedures, battery constraints, and emergency checklists using natural language—completely offline.

---

## System Architecture & Workflow

The project implements a modular software architecture divided into three fundamental layers:

1. **Data Ingestion & Extraction (Sensing):** Parses the technical PDF manual using `PyPDFLoader` with background OCR enabled (`extract_images=True`) to prevent the loss of data stored inside visual diagrams and tables.
2. **Vector Storage & Semantic Search (Retrieval):** Chunks text using a `RecursiveCharacterTextSplitter` (3000 chars size, 700 chars overlap) and maps segments into a dense vector space using the state-of-the-art multilingual **`BAAI/bge-m3`** embedding model. The mathematical indices are persisted locally via **ChromaDB**.
3. **Inference Core & Guardrails (Reasoning):** Leverages a local instance of **Llama 3** via **Ollama**. Creative hallucinations are strictly blocked by locking the model's temperature to `0.0` and enforcing a rigid prompt template contract.


## Engineering Highlights & Safety Guarantees

* **100% Privacy & System Autonomy (Independence Factor):** The entire pipeline operates locally on the host machine. It features zero dependencies on external APIs, internet connections, or third-party cloud architectures, making it ready for on-field edge operations.
* **CPU-Locked Computation:** Embedding mapping and semantic searches are computationally bounded to the host CPU (`device: cpu`), ensuring cross-platform portability without requiring dedicated NVIDIA CUDA GPU hardware layers.
* **Deterministic Hallucination Mitigation:** Armed with a strict context-anchoring fallback clause, the model acts purely as a rigid context processor. If the required technical spec is missing from the retrieved chunks, the system safely triggers a standardized refusal string (*"Non trovo questa informazione nel manuale"*).
* **Source Traceability & Verification:** Every factual claim generated by the local LLM is appended with custom metadata (`chunk_number` and native page numbers), allowing the human pilot to rapidly audit the source directly in the official manual.
* **Chronological Re-sorting:** To mitigate narrative fragmentation caused by the similarity search, retrieved chunks are chronologically sorted before building the prompt context window, restoring the document's native reading flow.

##  Getting Started (Local Deployment)

### 1. Prerequisites

Ensure you have **Python 3.10+** (or an Anaconda environment) and **Ollama** installed on your host system.

### 2. Set Up the Local LLM Core

Download and launch Ollama, then pull the target language model locally by running this command in your terminal:

```bash
ollama run llama3

```

*Note: Ollama must remain active as a background service listening on local port `11434` for the Python backend to communicate.*

### 3. Install Dependencies

Clone the repository, navigate into the project directory, and install the required software libraries:

```bash
pip install langchain langchain-community langchain-text-splitters langchain-chroma langchain-huggingface langchain-core langchain-ollama pypdf sentence-transformers chromadb

```

### 4. Execute the Interactive Assistant

Launch the primary application to query the manual using natural language:

```bash
python rag_drone_manual.py

```

### 5. Run Automated Verification Tests

Execute the testing suite to run the evaluation benchmark across 5 in-scope and 5 out-of-scope queries, auditing the deterministic output generation:

```bash
python test_pipeline.py

```

Check the generated `report_test_dji.txt` file to view the complete query-source-response log.
